# Feature: aa-002

## Source
- Requirement: `fr-aa-002` — Structured finding schema
- Detail: `specs/arc-auditing-spec/details/audit-agents.xml`

## Dependencies
- aa-001 (fan-out must exist before a schema can be enforced across its outputs)

## Acceptance Criteria
- [ ] `fr-aa-002-ac1` — Every finding MUST include: unique ID `A<axis>-<NNN>` (axis ∈ {1,2,3}, NNN zero-padded), severity ∈ {HIGH, MED, LOW}, one-line title, affected file paths with line references (when known), "observed" prose, "why it matters" paragraph, and up to 4 suggested resolutions.
- [ ] `fr-aa-002-ac2` — Each resolution includes a 1–5 word label, a description, and (when resolution corresponds to an editable artifact change) a `preview` diff string. Resolutions without editable equivalents MAY omit `preview`.
- [ ] `fr-aa-002-ac3` — When reviewer has a preference, first resolution prefixed `"(Recommended)"`; when no preference, no resolution carries the prefix.
- [ ] `fr-aa-002-ac4` — Severity is a reviewer-agent judgment call; specific HIGH/MED/LOW criteria live in each agent's system prompt (produced via arc-writing-skills), not in this spec.
