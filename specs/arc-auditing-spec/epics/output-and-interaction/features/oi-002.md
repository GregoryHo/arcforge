# Feature: oi-002

## Source
- Requirement: `fr-oi-002` — Triage UX (multi-select over HIGH findings with free-text pull-in)
- Detail: `specs/arc-auditing-spec/details/output-and-interaction.xml`

## Dependencies
- oi-001 (triage consumes Phase 2 output state)

## Acceptance Criteria
- [ ] `fr-oi-002-ac1` — First triage AskUserQuestion call sets `multiSelect: true`, `header: "Triage"`, and lists up to 4 HIGH-severity findings as `options` in stable order (axis-then-ID).
- [ ] `fr-oi-002-ac2` — When HIGH count > 4, sequential additional AskUserQuestion calls are issued (up to 4 HIGH each) until every HIGH finding has been presented exactly once.
- [ ] `fr-oi-002-ac3` — User's Other free-text containing finding IDs (format `A<axis>-<NNN>`, any severity including MED/LOW) MUST add those IDs to the Stage-2 resolution queue.
- [ ] `fr-oi-002-ac4` — MED and LOW findings MUST NOT appear in any triage call's `options` list; they are reachable only via the Other free-text channel.
