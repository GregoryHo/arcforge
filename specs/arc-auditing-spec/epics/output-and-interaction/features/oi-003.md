# Feature: oi-003

## Source
- Requirement: `fr-oi-003` — Resolution UX (batched per-finding AskUserQuestion with preview diffs)
- Detail: `specs/arc-auditing-spec/details/output-and-interaction.xml`

## Dependencies
- oi-002 (resolution consumes the triage queue)

## Acceptance Criteria
- [ ] `fr-oi-003-ac1` — Phase 4 batches N selected findings into AskUserQuestion calls of ≤4 questions each, using sequential calls until all N have been asked exactly once.
- [ ] `fr-oi-003-ac2` — Each question: `header` = finding ID (≤12 chars), `question` = title + compact observed summary, `options` = up to 4 resolutions, `multiSelect: false`.
- [ ] `fr-oi-003-ac3` — Resolutions corresponding to editable artifact changes MUST include a `preview` diff string; resolutions without an editable equivalent MAY omit `preview`.
- [ ] `fr-oi-003-ac4` — When the reviewer marked a recommended resolution, the first option's label starts with `"(Recommended)"`.
- [ ] `fr-oi-003-ac5` — User's Other free-text response MUST be accepted as a valid decision equivalent to the listed resolutions, without error.
